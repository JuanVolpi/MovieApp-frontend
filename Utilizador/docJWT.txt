 GERAÇÃO DE TOKENS (NO SERVIÇO DE UTILIZADOR)
------------------------------------------------
Endpoint: POST /api/users/login

Descrição da Modificação:
A rota de login foi alterada para, em caso de sucesso, gerar um token JWT.

Biblioteca Utilizada:
- PyJWT

Estrutura do Payload do Token (Corpo de Dados):
{
    "sub": "41",                  // Subject: O ID do utilizador (como string).
    "iat": 1749853011,            // Issued At: Timestamp de quando o token foi criado.
    "exp": 1749856611             // Expiration: Timestamp de quando o token expira (1 hora após a criação).
}

Mecanismo de Assinatura:
- O token é assinado digitalmente usando o algoritmo `HS256`.
- A assinatura utiliza uma `SECRET_KEY` centralizada, partilhada entre
  todos os microserviços através de um ficheiro `.env`.

Exemplo de Resposta de Login Atualizada:
{
    "message": "Login bem-sucedido!",
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": { ...dados do utilizador... }
}
